# <a name="user-adoption-guide"></a>用户采用指南

此信息适用于 Microsoft 搜索管理员、变更管理负责人和企业所有者。
  
为了推广并确保在整个组织内成功采用 Microsoft 搜索，我们建议你使用此四阶段流程。
  
## <a name="stage-1-stakeholders"></a>第 1 阶段：利益干系人

为了确保无缝推广并提高用户对 Microsoft 搜索的采用率，请确定可以帮助支持和推动该过程的人员。
  
该表显示了团队层次结构以及每个人的贡献。 
  
 **用户采用团队角色和职责**
  
|**角色**|**职责**|**部门**|
|:-----|:-----|:-----|
|执行发起人  <br/> |了解 Microsoft 搜索对贵组织的高级愿景和价值  <br/> 为 IT/业务团队提供领导力支持  <br/> 分配资源  <br/> 直接与员工沟通  <br/> |行政领导  <br/> |
|企业所有者  <br/> |确定产品和用户的应用场景  <br/> 确定部门和组织的成功指标  <br/> |任何  <br/> |
|变更管理团队  <br/> |确保产品价值易于理解  <br/> 创建推广计划  <br/> |IT  <br/> |
|支持者  <br/> |向同事和团队宣传产品  <br/> |任何（人员）  <br/> |
|搜素管理员  <br/> |确定用户采用核心团队成员  <br/> 管理 Microsoft 搜索，包括体验和持续管理的外观和感觉  <br/> 推动讨论以展示 Microsoft 搜索的价值  <br/> |IT  <br/> |
|编辑人员  <br/> |在管理门户中创建内容  <br/> |IT/任何  <br/> |
   
## <a name="stage-2-scenarios"></a>第 2 阶段：应用场景

为了确保用户采用，确定和定义最终用户的应用场景至关重要。 这些应包括如何使用 Microsoft 搜索查找所需信息、回答问题以及更快完成任务的实际示例。 为了评估推广和用户采用的成功与否，在开始之前确定需要使用的指标也很重要。
  
 **了解用户的需求**
  
在寻找信息和获得答案时，了解用户的需求和痛点非常重要。 你可以使用定量方法（如 Office 365 和 SharePoint 搜索使用的数据）或定性方法（如调查和用户焦点组）来获取此数据。 这将帮助你确定 Microsoft 搜索可以提供价值的时间、地点和方式。
  
对于整个组织中的各种角色和团队，请考虑：
  
- 搜索信息时存在的用户痛点。
    
- 经常使用的应用、工具、网站和其他信息来源。
    
- 用户在搜索信息时希望看到的内容。
    
- Microsoft 搜索为贵组织带来的好处。 有关详细信息，请参阅[为何使用 Microsoft 搜索](why-microsoft-search.md)。
    
选择对用户产生最大积极影响的用户应用场景。
  
 **确定成功指标**
  
为了帮助你实现推广目标并为组织带来积极影响，在开始之前确定关键成功指标至关重要。 确保目标简单且可衡量并定期评估绩效，以便在需要时快速做出响应。
  
1. **变更管理：** 确定如何衡量对 Microsoft 搜索的认知以及与之相关的通信是否成功。 
    
2. **使用情况：** 你可以利用见解仪表板中的数据，包括每日和每月活跃用户（DAU 和 MAU）、观看次数最多的内容类型以及用于衡量一段时间内使用情况的热门搜索关键字。 有关详细信息，请参阅[使用见解仪表板](get-insights.md)。
    
3. **用户满意度：** 使用从调查和用户焦点组收集的数据来确定你的基准。 净满意度 (NSAT) 和净推荐值 (NPS) 也可以提供有用的评估。 无论你使用哪种方法，都必须确保一致，以便及时了解影响。 
    
4. **其他评估：** 使用支持请求以及搜索所用时间的历史和/或当前数据。 
    
## <a name="stage-3-readiness"></a>第 3 阶段：准备就绪

投入时间来准备推广 Microsoft 搜索将对你的组织产生积极影响。 
  
 **产品和内容准备情况**
  
管理员直接影响最终用户的搜索体验，并负责确定 Microsoft 搜索如何查找组织。 这包括选择要向用户显示的结果类型。 通过打开或关闭文档、网站、人员、群组或对话的结果，可对此进行控制。
  
有关详细信息，请参阅[设置 Microsoft 搜索](set-up-microsoft-search.md)。
  
编辑人员是主题专家，他们负责在其专业领域内帮助管理书签和问答。 编辑人员应创建相关内容来满足你之前确定的用户应用场景的要求。
  
有关详细信息，请参阅[规划内容](plan-your-content.md)。
  
 **登录**
  
Microsoft 搜索用户必须使用工作或学校帐户登录必应。 若要让用户尽可能无缝地登录：
  
- 确保正确设置 Azure Active Directory 配置，让少数用户尝试登录。 有关详细信息，请参阅[测试单点登录](test-single-sign-on.md)。
    
- 将公司管理的设备的默认浏览器设为必应。 有关详细信息，请参阅[设置默认浏览器](set-default-browser.md)。
    
- 通过组策略为 Microsoft Edge 和 Chrome 用户部署搜索扩展。 有关详细信息，请参阅[管理门户工具](admin-portal-tools.md)。
    
 **草拟通信**
  
使用实际应用场景向用户介绍 Microsoft 搜索及其如何帮助他们节省时间。 你的变更管理团队可以帮助制定推广策略，并准备好宣传材料和营销活动，让最终用户做好准备。
  
开始为你的组织规划通信和活动：
  
- 在醒目位置投放横幅、海报或宣传册
    
- 面对面的产品演示或展位
    
- 本地和远程团队的在线活动
    
- 让执行赞助商主持员工大会或发送公告电子邮件
    
- 共享有关使用 Microsoft 搜索的自助文档
    
所有通信都应该向用户介绍如何登录必应以查找工作结果。
  
 **接洽合作伙伴团队**
  
作为 Microsoft 365 不可或缺的一部分，Microsoft 搜索符合 GDPR 和 Tier-C 指南的要求。 根据贵组织的要求，可能需要与隐私和安全团队共享有关 Microsoft 搜索的信息。 有关详细信息，请参阅 [Microsoft 搜索安全性](security.md)。
  
确保你的技术支持了解 Microsoft 搜索的特性和功能以及用户登录的要求，以便他们能够更加高效地为用户提供支持。 有关详细信息，请参阅[使用 Microsoft 搜索](use/about-microsoft-search.md)和[常见问题解答](faqs.md)。
  
 **试点用户评估**
  
从一小组用户开始，评估 Microsoft 搜索的影响并收集早期反馈。 这让你有时间更改内容、更新通信并为广泛的推广设定期望。 搜索超级用户或早期采用者是很好的选择。
  
为了帮助查找试点用户：
  
- 与组织内的业务发起人合作
    
- 标识早期采用者组
    
- 标识执行大量企业搜索的组
    
- 标识其他地理位置的用户，包括国际用户
    
让试点用户使用 Microsoft 搜索中的反馈选项或通过简单调查来提供反馈意见。
  
可询问的调查问题： 
  
- 你是否了解如何登录和使用 Microsoft 搜索？
    
- 它是否按预期工作？
    
- 你喜欢哪些功能？
    
- 你能找到工作结果吗？
    
- 工作结果有用吗？
    
- 有关 Microsoft 搜索的通信是否清晰易懂？
    
现在是开始评估在第 2 阶段确定的指标效果的好时机。
  
当你开始试点阶段时，请考虑与更广泛的组织分享 Microsoft 搜索的最新资讯。
  
## <a name="stage-4-roll-out"></a>第 4 阶段：推广

每个组织对产品推广有着不同的看法。 以下是一些需要考虑的事项。
  
 **广泛部署**
  
当你准备就绪后，允许组织内的所有用户访问 Microsoft 搜索，或使用分阶段方法以较慢速度进行推广。 
  
 **培养意识**
  
在组织内激发员工对 Microsoft 搜索的兴趣：
  
- 争取让试点用户提供点对点协助和专业知识。
    
- 跟进你在第 3 阶段规划的通信和活动。
    
 **持续通信**
  
在最初的几周和几个月中，保持通信节奏以鼓励与 Microsoft 搜索进行互动。 这可以让用户了解情况并吸引新用户。 每次通信都应解释 Microsoft 搜索为何对他们的工作有所帮助。
  
发送持续通信内容，突出显示要尝试的功能和搜索以及用户成功案例。
  
 **收集反馈并改进**
  
为用户提供支持，通过聆听他们的想法并根据其反馈采取相关措施，让他们感觉受到重视。
  
使用见解仪表板了解产品和内容的效果。 利用此信息更新书签、问答和其他通信。 这还可以帮助你：
  
1. 提高内容质量：审核标题、说明和 URL、优化关键字、删除过时信息以及添加新信息
    
2. 确定最常使用或不经常使用的功能
    
3. 展开调查、焦点组和反馈会话，以了解用户对 Microsoft 搜索的看法、提高采用率并确定改进领域
    
教育和反馈是一个迭代过程。 当用户采用 Microsoft 搜索时，可能需要更改内容和通信。

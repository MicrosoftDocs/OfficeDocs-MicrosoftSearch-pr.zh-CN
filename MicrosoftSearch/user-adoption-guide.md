# <a name="user-adoption-guide"></a>用户采用指南

此信息适用于 Microsoft 搜索管理员、 更改管理负责人和业务所有者。
  
要推出并确保您的组织内的成功采用 Microsoft 搜索，我们建议此四阶段过程。
  
## <a name="stage-1-stakeholders"></a>阶段 1： 利益干系人

若要确保无缝推出和增加的 Microsoft 搜索用户应用，确定支持和驱动器过程可以帮助的人员。
  
下表显示的团队层次结构和每个人的贡献。 
  
 **用户应用工作组角色和职责**
  
|**角色**|**职责**|**Department**|
|:-----|:-----|:-----|
|执行发起人  <br/> |了解高级构想和 Microsoft 搜索为组织的值  <br/> 为提供高领导层支持 IT / 业务团队  <br/> 分配资源  <br/> 直接与员工进行通信  <br/> |Executive 领导  <br/> |
|企业所有者  <br/> |确定产品和用户方案  <br/> 确定为部门和组织成功指标  <br/> |任何  <br/> |
|更改管理小组  <br/> |确保产品值是易于理解  <br/> 创建部署规划  <br/> |IT  <br/> |
|拥护者  <br/> |宣传同事和团队产品  <br/> |任何 （人员）  <br/> |
|搜索管理员  <br/> |标识用户采用核心团队成员  <br/> 管理 Microsoft 搜索，包括体验和日常管理的外观  <br/> 驱动器讨论显示 Microsoft 搜索的值  <br/> |IT  <br/> |
|编辑器  <br/> |在管理门户中创建内容  <br/> |任何 IT /  <br/> |
   
## <a name="stage-2-scenarios"></a>阶段 2： 方案

若要确保用户应用，它非常重要来确定和定义最终用户方案。这些应包括现实世界的人员如何使用 Microsoft 搜索以查找所需，回答问题，并更快地完成任务的信息的示例。为了衡量您推广成功和用户采用也很重要来确定开始之前，您将使用的度量数据。
  
 **了解您的用户需要并希望**
  
当谈到查找信息并获得问题的答案时，了解您的用户需求和难题是密钥。您可以使用定量方法，如数据从 Office 365 和 SharePoint 搜索使用率或定性方法，如调查和用户焦点组来获取此数据。这将帮助您确定何时 Microsoft 搜索位置和方式可以提供值。
  
有关各种角色和团队整个组织，请考虑：
  
- 现有用户困难点时搜索的信息。
    
- 常用的应用程序、 工具、 网站和其他信息来源。
    
- 哪些用户想要查看当谈到搜索信息。
    
- 如何 Microsoft 搜索好处适用于您的组织。有关详细信息，请参阅[为什么 Microsoft 搜索](why-microsoft-search.md)。
    
选择会为您的用户的最大的正值影响的用户方案。
  
 **确定成功指标**
  
可帮助您实现推出目标和显示为您的组织，积极的影响很重要开始之前确定关键成功指标。简单和可测量保留您的目标，并评估定期的性能，以便您能够快速作出响应，如果需要。
  
1. **变更管理：** 确定您将如何衡量的 Microsoft 搜索的知晓和有关该产品的通信的成功。 
    
2. **使用率：** 您可以利用数据获取最视图和顶部的内容类型中包括每天和每月活动用户 （DAU 和 MAU） 见解仪表板，搜索关键字随着时间的推移衡量使用情况。有关详细信息，请参阅[使用见解仪表板](get-insights.md)。
    
3. **用户满意度：** 使用从调查和用户焦点组来确定您基准收集的数据。Net 满意度 (NSAT) 和 Net 者均分数 (NPS) 还可以提供有用的度量值。无论该方法使用，保持一致，以便您可以看到随着时间的影响。 
    
4. **其他度量值：** 将历史和/或当前数据用于支持请求和搜索花费的时间量。 
    
## <a name="stage-3-readiness"></a>阶段 3： 准备

投资中准备您的 Microsoft 搜索推出的时间将导致为您的组织积极的影响。 
  
 **产品和内容的准备情况**
  
管理员直接影响最终用户搜索体验，并确定 Microsoft 搜索组织的外观。这包括选择您要向用户显示的结果的类型。此进行控制的打开或关闭的文档、 网站、 人员、 组或对话的结果。
  
有关详细信息，请参阅[设置 Microsoft 搜索](set-up-microsoft-search.md)。
  
编辑器是帮助管理的专业技能自己区域内的书签和 Q&As 行业专家那里获得。编辑器应创建地址前面提到的用户方案的内容。
  
有关详细信息，请参阅[规划内容](plan-your-content.md)。
  
 **登录**
  
Microsoft Search 用户必须登录到 Bing 工作或学校帐户。若要使登录尽可能流畅的用户：
  
- 确保已正确设置 Azure Active directory 配置，必须将一些用户尝试登录。有关详细信息，请参阅[测试单一登录](test-single-sign-on.md)。
    
- 设置 Bing 公司管理设备的浏览器默认值。有关详细信息，请参阅[设置默认浏览器](set-default-browser.md)。
    
- 部署边缘服务器和部件版式通过组策略的用户的搜索扩展。有关详细信息，请参阅[管理门户工具](admin-portal-tools.md)。
    
 **草稿通信**
  
使用您现实世界的方案以告知用户有关 Microsoft 搜索以及它如何帮助这些节省时间。创建推出策略和辅助资料和市场活动准备最终用户准备情况，可帮助您更改管理小组。
  
开始规划通信和您的组织的事件：
  
- 横幅、 海报、 或传单放在主要位置
    
- 面对面产品演示或收听间
    
- 本地和远程团队 online 事件
    
- 具有您执行发起人主机城镇大厅会议或发送通知电子邮件
    
- 有关使用 Microsoft 搜索共享自助文档
    
所有通信应都告知用户有关如何他们可以登录到 Bing 查找工作结果。
  
 **使用合作伙伴团队**
  
作为 Microsoft 365 的组成部分，Microsoft 搜索符合 GDPR 和层 C 指南。根据您的组织，可能需要与您的隐私和安全团队共享 Microsoft 搜索有关的信息。有关详细信息，请参阅[Microsoft 搜索功能的安全性](security.md)。
  
确保帮助台知道有关的特性和功能的 Microsoft 搜索，以及用户需求登录，以便他们可以更有效地支持用户。有关详细信息，请参阅[使用 Microsoft 搜索](use/about-microsoft-search.md)和[常见问题解答](faqs.md)。
  
 **评估与试生产用户**
  
开始一小组用户评估 Microsoft 搜索的影响，并收集早期的反馈。这样，以更改内容、 更新您的通信，并设置对三大推出期望的时间。搜索超级用户或前期应用是非常适用此。
  
若要帮助查找试点用户：
  
- 与组织内业务部门合作
    
- 确定前期应用一组
    
- 确定组执行大量企业级搜索
    
- 确定其他地理位置，包括国际中的用户
    
提出试生产用户提供使用反馈选项，在 Microsoft 搜索或通过简单调查的输入。
  
要询问的调查问题： 
  
- 您是否知道如何登录和使用 Microsoft 搜索？
    
- 它工作预期？
    
- 您喜欢哪些功能？
    
- 已能够找到工作结果？
    
- 已工作结果有用？
    
- 有关 Microsoft 搜索清楚明了通信？
    
这是一个极好开始测量的标识在第 2 阶段的度量数据有效性。
  
启动试验阶段时，请考虑使用较大组织共享 Microsoft 搜索快速浏览。
  
## <a name="stage-4-roll-out"></a>第 4 阶段： 推出

每个组织想到的产品的推广不同。以下是一些要考虑的事项。
  
 **广泛部署**
  
准备就绪后，允许您的组织对 Microsoft Search 部门中的所有用户或使用分阶段的方法缓慢推出的详细信息。 
  
 **形成认知度**
  
生成有关 Microsoft 搜索组织中的兴奋：
  
- 登记试点用户提供对等帮助和专业技能。
    
- 跟进上的通信和您计划阶段 3 中的事件。
    
 **正在进行通信**
  
第一个几周和月，维护节奏通信鼓励与 Microsoft 搜索的项目。此保留通知用户，并且接合新用户。每个通信应解释为什么 Microsoft 搜索可能非常有帮助与其工作。
  
功能和搜索以尝试运行，请突出显示的正在进行通信以及用户成功案例发送。
  
 **收集反馈以及改进**
  
支持用户，并让他人了解您通过其侦听和对其反馈操作所关注的原因。
  
使用见解仪表板查看如何执行产品和内容。利用此信息来更新书签、 Q&As 和其他通信。这将帮助您：
  
1. 改进内容质量： 查看标题、 说明和 Url，优化关键字、 删除过时的信息，并添加新的信息
    
2. 确定哪些功能是最常 — 或不经常 — 使用
    
3. 进行调查、 焦点组和反馈会话了解用户的 Microsoft 搜索的想法，提高应用率，并确定改善区域
    
教育和反馈是迭代的过程。可能需要进行更改对内容和沟通作为您的用户采用 Microsoft 搜索。

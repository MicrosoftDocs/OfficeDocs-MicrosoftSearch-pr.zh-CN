# <a name="user-adoption-guide"></a>用户采用指南

此信息适用于 Microsoft 搜索管理员、更改管理领导者和业务所有者。
  
若要在整个组织中部署并确保成功采用 Microsoft 搜索，我们建议采用这四个阶段的过程。
  
## <a name="stage-1-stakeholders"></a>第1阶段：利益干系人

为确保无缝部署并提高用户对 Microsoft 搜索的采用，请确定可帮助支持和推动此过程的人员。
  
该表显示团队层次结构以及每个人所贡献的内容。 
  
 **用户采用团队角色和职责**
  
|**角色**|**负责**|**Department**|
|:-----|:-----|:-----|
|执行发起人  <br/> |了解组织的 Microsoft 搜索的高级别远景和价值。  <br/> 为 IT/业务团队提供领导能力支持。  <br/> 分配资源。  <br/> 与员工直接沟通。  <br/> |执行官领导  <br/> |
|企业所有者  <br/> |确定产品和用户方案。  <br/> 确定部门和组织的成功指标。  <br/> |任意  <br/> |
|更改管理团队  <br/> |确保产品价值易于理解。  <br/> 为推出创建计划。  <br/> |IT  <br/> |
|拥护  <br/> |将产品 Evangelize 到对等方和团队。  <br/> |Any （教职员工）  <br/> |
|搜索管理员  <br/> |确定用户采用核心团队成员。  <br/> 管理 Microsoft 搜索，包括体验和持续管理的外观和感觉。  <br/> 推动讨论以显示 Microsoft Search 的价值。  <br/> |IT  <br/> |
|Editors  <br/> |在 Microsoft 365[管理中心](https://admin.microsoft.com)中创建内容。  <br/> |IT/任何  <br/> |
   
## <a name="stage-2-scenarios"></a>第2阶段：方案

为了确保用户采用，确定和定义最终用户方案至关重要。 这些示例应包括以下内容的真实示例：用户如何使用 Microsoft Search 查找所需的信息，回答问题，并更快地完成任务。 为了衡量您的部署和用户采用是否成功，在开始之前，确定您将使用的指标也是非常重要的。
  
### <a name="understand-what-your-users-need-and-want"></a>了解用户需要和需要的内容
  
在查找信息和获取答案时，了解您的用户需求和痛点是关键。 您可以使用定量方法，如来自[Office 365](https://Office.com)和[SharePoint](https://products.office.com/sharepoint/collaboration)搜索使用的数据，或者定性方法（如调查和用户焦点组）来获取这些数据。 这将帮助您确定 Microsoft Search 可以在什么时间、何处以及 Microsoft Search 如何提供值。
  
对于组织中的各种角色和团队，请考虑以下事项：
  
- 搜索信息时的现有用户痛点。
    
- 频繁使用的应用程序、工具、网站和其他信息源。
    
- 在搜索信息时，用户想要查看的内容。
    
- Microsoft 搜索对您的组织的好处。 有关详细信息，请参阅[Microsoft 搜索概述](overview-microsoft-search.md)。
    
选择将对你的用户产生最大积极影响的用户方案。
  
 ### <a name="identify-success-metrics"></a>确定成功指标
  
为了帮助您满足您的部署目标并向您的组织显示积极影响，在开始之前确定关键成功指标非常关键。 将您的目标保持简单且可衡量，并定期评估性能，以便在需要时可以快速做出响应。
  
1. **更改管理：** 确定您将如何衡量 Microsoft 搜索的认知度以及有关它的通信是否成功。 
    
2. **用法：** 您可以利用 Insights 仪表板中的数据，其中包括每日和每月活动用户（DAU 和 MAU）、获取最多视图的内容类型，以及搜索关键字以根据时间测量使用情况的主要搜索关键字。 有关详细信息，请参阅[Microsoft Search Insights 仪表板报告](get-insights.md)。
    
3. **用户满意度：** 使用从调查和用户焦点组中收集的数据来确定您的基准。 Net 满意度（NSAT）和 Net 净比分（NPS）也可以提供有用的度量。 无论您使用哪种方法，保持一致，以便您可以看到一段时间内的影响。 
    
4. **其他度量：** 使用历史和/或当前数据获取支持请求以及搜索所用的时间量。 
    
## <a name="stage-3-readiness"></a>第3阶段：准备情况

为 Microsoft Search 部署做准备的投资时间将导致您的组织产生积极影响。 Microsoft 提供了<a href="https://resources.techcommunity.microsoft.com/microsoft-search-in-bing-adoption-kit/">采用工具包</a>，其中包含规划指南、可自定义的电子邮件通知、调查模板、执行演示等。 还有<a href="https://resources.techcommunity.microsoft.com/microsoft-search/#adoption">Microsoft 搜索资源中心</a>，IT 实施者可以在其中找到其他培训和采用资源，旨在帮助 Microsoft 搜索首次部署成功。
  
### <a name="product-and-content-readiness"></a>产品和内容准备情况
  
管理员直接影响最终用户的搜索体验，并确定 Microsoft 搜索对组织的外观。 这包括选择要向用户显示的结果类型。 这些结果可能包含链接（也称为书签）到文档、网站、人员、组、位置、对话和其他工作区资源。
  
有关详细信息，请参阅[设置 Microsoft Search](setup-microsoft-search.md)。
  
编辑者是一种主题专家，可帮助管理书签和 Q 在其专业领域中&。 编辑器应创建用于解决您之前标识的用户方案的内容。
  
有关详细信息，请参阅[规划内容](plan-your-content.md)。
  
### <a name="signing-in"></a>登录
  
Microsoft Search 用户必须使用工作或学校帐户登录[Bing](https://Bing.com) 。 若要尽可能无缝登录用户，请执行以下操作：
  
- 若要确保正确设置了[Azure Active Directory](https://docs.microsoft.com/azure/active-directory/)配置，请让少数用户尝试登录。 有关详细信息，请参阅[Test single sign-on](test-single-sign-on.md)。
    
- 将公司托管设备的浏览器默认值设置为 Bing。 有关详细信息，请参阅[Set default browser](set-default-browser.md)。
    
- 通过组策略为[Microsoft Edge](https://www.microsoft.com/windows/microsoft-edge)和[Google Chrome](https://www.google.com)用户部署搜索扩展。 有关详细信息，请参阅[Microsoft 搜索概述](overview-microsoft-search.md)。
    
### <a name="draft-communications"></a>草稿通信
  
使用您的真实应用场景告诉用户 Microsoft 搜索以及它将如何帮助他们节省时间。 您的更改管理团队可帮助创建首展策略，并为最终用户准备就绪准备辅助资料和市场活动。
  
开始规划组织的通信和事件：
  
- 将横幅、海报或 leaflets 放置在突出的位置。
    
- 人员内产品演示或 booths。
    
- 本地团队和远程团队的联机事件。
    
- 让你的执行发起人主持城镇大厅会议或发送通知电子邮件。
    
- 共享有关使用 Microsoft 搜索的自助式文档。
    
所有通信都应告知用户如何登录[Bing](https://Bing.com)以查找工作结果。
  
### <a name="engage-partner-teams"></a>合作合作伙伴团队
  
作为[microsoft 365](https://www.microsoft.com/microsoft-365)不可或缺的一部分，microsoft SEARCH 遵循 GDPR 和第3层指南。 根据您的组织，可能需要与您的隐私和安全团队共享有关 Microsoft 搜索的信息。 有关详细信息，请参阅[Bing 中的 Microsoft Search 安全性和隐私](security-for-search.md)。
  
确保您的技术支持人员了解 Microsoft Search 的特性和功能，以及用户对登录的要求，以便它们可以更有效地支持用户。 有关详细信息，请参阅[在 microsoft 365 和 faq 中了解 Microsoft Search](https://support.office.com/article/Learn-about-Microsoft-Search-in-Microsoft-365-b8bf5a2c-7515-40a9-9a6a-b8ed382c86bc) 。 [](faqs.md)
  
### <a name="evaluate-with-pilot-users"></a>试用用户评估
  
从一小组用户开始，以评估 Microsoft 搜索的影响并收集早期反馈。 这为您提供了更改内容、更新通信和设置广泛部署的预期的时间。 搜索高级用户或早期采用者非常适合这样做。
  
若要帮助查找试用用户：
  
- 与组织内的企业赞助商合作。
    
- 确定一组早期采用者。
    
- 确定执行大量企业级搜索的组。
    
- 标识其他地理位置（包括国际）中的用户。
    
请让试点用户使用 Microsoft Search 或简单调查中的反馈选项提供输入。
  
调查要询问的问题： 
  
- 您是否了解如何登录并使用 Microsoft Search？
    
- 它是否按预期工作？
    
- 您喜欢哪些功能？
    
- 您能够找到工作结果吗？
    
- 工作成果是否有帮助？
    
- Microsoft Search 是否清楚和可理解的通信是什么？
    
若要开始衡量在第2阶段中确定的指标的有效性，这是一项非常好的时间。
  
在您开始引导阶段时，请考虑与更广泛的组织共享 sneak 查看 Microsoft Search。
  
## <a name="stage-4-roll-out"></a>第4阶段：推出

每个组织都认为产品的部署方式不同。 以下是要考虑的几个事项。
  
### <a name="broad-deployment"></a>广泛部署
  
准备就绪后，可以允许组织中的所有用户访问 Microsoft Search，也可以使用分阶段的方法来更缓慢地推出。 
  
### <a name="generate-awareness"></a>生成感知
  
在您的组织中构建关于 Microsoft 搜索的惊喜：
  
- 登记试点用户以提供对等协助和专业技能。
    
- 按照您在第3阶段中规划的通信和事件进行操作。
    
### <a name="ongoing-communication"></a>持续通信
  
在前几个星期和月中，维护节奏的通信，以鼓励与 Microsoft 搜索的接洽。 这将使用户有通知并参与新用户。 每个通信应说明 Microsoft Search 对其工作有帮助的原因。
  
发送正在进行的通信，以突出显示功能和搜索以尝试，以及用户成功情景。
  
### <a name="gather-feedback-and-improve"></a>收集反馈并提高
  
支持用户，并让他们知道你是通过倾听他们的反馈来处理的。
  
使用 "见解" 面板查看产品和内容的执行情况。 利用此信息更新书签、Q&As 和其他通信。 这还将帮助您：
  
- 改进内容质量：查看标题、说明和 Url、优化关键字、删除过时信息和添加新信息。
    
2. 确定哪些功能最常用或不常用。 
    
3. 执行调查、关注团队和反馈会话，以了解用户对 Microsoft 搜索、提高采用和确定改进方面的看法。
    
教育和反馈是一个反复的过程。 在用户采用 Microsoft Search 时，可能需要对内容和通信进行更改。
